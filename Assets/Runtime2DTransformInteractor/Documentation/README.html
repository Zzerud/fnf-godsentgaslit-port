Runtime 2D Transform Interactor

Overview

Runtime 2D Transform Interactor is a package that allows you to interact with Transform components in 2D easily. It displays a rectangle around the target object, and you can interact with it to move, scale and rotate your Transform components.

Features

Here are all the features currently supported by this package:

  • Changing the position of the transform : When the mouse is on the transform collider, you can move the transform around by simply clicking and dragging it.
  • Changing the scale of the transform : Clicking and dragging a corner or the edge of the bounding rectangle allows you to rescale the transform on one (for the lines) or 2 (for the corners) axis.
  • Changing the sclae of the transform while keeping the same aspect ratio : You can scale the transform and keep the same aspect ratio if you want to. This feature can either always be activated, activated only when the user presses some customizable hotkeys, or deactivated.
  • Changing the rotation of the transform : You can rotate the transform by clicking and dragging the rotation gizmo. This feature is deactivable.
  • Changing the mouse cursor : You can choose the mouse cursor you want to use when the cursor is over the collider of the target, or the corners, edges, and rotation gizmo of the bounding rectangle. Example mouse cursors are available in the package. This feature is deactivable.
  • Multi-selecting objects : You can select multiple objects that have a TransformInteractor script to move them together using customizable hotkeys. This feature is deactivable.
  • Customizing the bounding rectangle : Here are the elements of the bounding rectangle that you can customize:
    • The corners sprite
    • The corners color
    • The rotation gizmo sprite
    • The rotation gizmo color
    • The rectangle edges color
  • Adjusting the size of the bounding rectangle elements to the camera zoom : The corners, edges, and rotation gizmo of the bounding rectangle will always appear the same size on the screen when your camera zooms. This feature requires an orthographic camera as it is based on the camera orthographicSize parameter. The size of each elements can be set by the user. This feature is deactivable.

How To Use

First add the TransformInteractorController prefab in the Package folder to your scene. The prefab already has some default values set, and that is why I recommend using it. Alternatively, you can find the TransformInteractorController script in the Package/Scripts folder, and attach it to any GameObject you want in your scene.

Then, set the parameters you want on the TransformInteractorController script. Those parameters are:

  • Bounding Rectangle Prefab : the prefab used for the bounding rectangle that allows interactions. It shouldn't be changed.
  • Corner Sorting Layer : the name of the sorting layer to use for the corners of the bounding rectangle. Should be over the line sorting layer.
  • Line Sorting Layer : the name of the sorting layer to use for the edges of the bounding rectangle. Should be under the corner and rotation gizmo sorting layer.
  • Rotation Gizmo Sorting Layer : the name of the sorting layer to use for the rotation gizmo. Should be over the line sorting layer.
  • Max Z displacement : the distance between the corners and the TransformInteractor element. Corners and lines of the bounding rectangle are in front of the TransformInteractor element (to avoid having overlapping 2D colliders).
  • Corner Sprite : the sprite to use for the corners. If null, the default icon will be used (located in Package/Icons/Gizmos/circle.png).
  • Corner Color : the color to give to the corner sprite.
  • Line Color : the color of the edges of the bounding rectangle.
  • Rotation Sprite : the sprite to use for the rotation gizmo. If null, the default icon will be used (located in Package/Icons/Gizmos/rotate.png).
  • Rotation color : the color to give to the rotation gizmo sprite.
  • Allow Multi Selection : if true, the user will be able to multi select elements with a TransformInteractor script by pressing the hotkey and clicking on them.
  • Multi Selection Keys : the hotkeys that the user has to press to multi select objects.
  • Unselect When Clicking Outside : if true, the objects will be unselected when the user clicks outside a collider.
  • Allow Rotation : if true, the rotation gizmo will be displayed. Otherwise, it is not possible to rotate the object.
  • Change Cursor : if true, mouse cursor will be changed when hovering the different interactive parts of the bounding rectangle.
  • Default Cursor : the default mouse cursor to use.
  • Move Cursor : the cursor displayed when hovering the object collider.
  • Resize Top Bottom Cursor : the cursor displayed when hovering the bottom and top edges of the bounding rectangle. Will be used for other parts if the transform is rotated.
  • Resize Top Left Bottom Right Cursor : the cursor displayed when hovering the top left and bottom right corners. Will be used for other parts if the transform is rotated.
  • Resize Left Right Cursor : the cursor displayed when hovering the left and right edges of the bounding rectangle. Will be used for other parts if the transform is rotated.
  • Resize Bottom Left Top Right Cursor : the cursor displayed when hovering the bottom left and top right corners. Will be used for other parts if the transform is rotated.
  • Rotate Cursor : the cursor displayed when hovering the rotation gizmo.
  • Adjust Size To Zoom : if true, size of the bounding rectangle elements will be the same on screen, whatever the orthographicSize of the camera is.
  • Main Camera : the main camera. It should be an orthographic camera as the orthographicSize property will be used to calculate the element size
  • Default Corner Width : the default width of the corner sprite. A default value that often fits my needs is 7.
  • Default Corner Height : the default height of the corner sprite. A default value that often fits my needs is 7.
  • Default Line Width : the default width of the edges of the bounding rectangle. A default value that often fits my needs is 5.
  • Default Rotation Width : the default width of the rotation gizmo sprite. A default value that often fits my needs is 7.
  • Default Rotation Height : the default height of the rotation gizmo sprite. A default value that often fits my needs is 7.
  • Always Preserve Aspect Ratio : if true, aspect ratio will always be preserved when scaling an object.
  • Preserve ASpect Ratio On Key Hold : if true, aspect ratio will be preserved only when a hotkey is hold.
  • Aspect Ratio Keys : the hotkeys to hold for keeping aspect ratio when scaling objects.

Finally, add a TransformInteractor script to any GameObject you want to be able to interact with. If the GameObjects doesn't have a BoxCollider2D already, one will be added. Resize it as you want as the bounding rectangle will match the BoxCollider2D size.
